Class FB.App

This module provides application settings, states (session states, permissions, etc.) and auth related methods (connect, logout)

Properties

session - static object

current session. When session exists, it is standard session structure that contains session_key, uid, session_secret, expire_time Note we use undefined to signal that we don't know yet

settings - static object

A dictionary of advanced facebook settings. Currently supported key/value are:

status - static string

current connect status. Note we use undefined to signal that we don't know yet

monitor

static void monitor ( name , callback , no_sync_callback , auto_unsubscribe )
Watch for changes in FB.App.
Parameters:
name <string> Name of event. possible values are 'session', 'status', 'init'
callback <function> A callback function. arguments may be passed to the callback. If the callback function returns true, the event will be unsubscribed.
no_sync_callback <bool> By default, the callback will be invoked immediately before the event is fired, unless this parameter specified a true value
auto_unsubscribe <bool> By default, the callback will be invoked whenever the event fires. However, if this parameter value is true, the callback will be unsubscribed from the event after it is fired once.
Returns: void
