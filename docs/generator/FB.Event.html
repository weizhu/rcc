private Class FB.Event

Basic Event system

Methods

add

static void add ( obj , name , callback )
Subscribe to an event on the given object Example: FB.Event.add(FB.App, 'session', function(session) { if (session) { // Yeah!. User is connected ... return true; // I don't need to listen anymore } // Hmm. no session. Keep listening }
Parameters:
obj <object> object that exposes the named event
name <string> Name of event
callback <function> A callback function. arguments may be passed to the callback. If the callback function returns true, the event will be subscribed.
Returns: void

name <string> Name of event
args0, <object> .., argN arguments to be passed to callback functions that subscribed to the event
Returns: void

monitor

static void monitor ( obj , name , callback , no_sync_callback , auto_unsubscribe )
Watch for changes
Parameters:
obj <object> object that exposes the named event
name <string> Name of event.
callback <function> A callback function. arguments may be passed to the callback. If the callback function returns true, the event will be unsubscribed.
no_sync_callback <bool> By default, the callback will be invoked immediately before the event is fired, unless this parameter specified a true value
auto_unsubscribe <bool> By default, the callback will be invoked whenever the event fires. However, if this parameter value is true, the callback will be unsubscribed from the event after it is fired once.
Returns: void

remove

static void remove ( obj , name , callback )
Unsubscribe to an event on the given object. Note you can also unsubscribe an event by simply return true in the callback function provided to FB.Event.subscribe. Example: FB.Event.add(FB.App, 'connectState', mycallback); ... // Dont' need to subscribe to the event anymore FB.Event.remove(FB.App, 'connectState', mycallback);
Parameters:
obj <object> object that exposes the named event
name <string> Name of event
callback <function> previous callback function passed to FB.Event.add
Returns: void
